Troubleshooting & Best Practices

Comprehensive guide to resolving common issues and optimizing your AbhiStat experience.

Quick Help: Use Ctrl+F (Cmd+F on Mac) to quickly search for specific error messages or issues on this page.

6.1 Common Issues & Error Resolution

Authentication & Access Issues

Error: "Invalid email or password"

Symptoms: Login fails with error message during standard login

Cause: Incorrect credentials or account doesn't exist

Solution:

  1. Verify your email address is correct
  2. Check if Caps Lock is enabled
  3. Try resetting your password if you're an external user
  4. For Abhitech internal users, contact your system administrator

Prevention: Use a password manager to store credentials securely

Error: "Invalid Google token"

Symptoms: Google OAuth login fails

Cause: Expired or invalid Google authentication token

Solution:

  1. Clear your browser cookies and cache
  2. Sign out of your Google account and sign back in
  3. Try using an incognito/private browser window
  4. Disable browser extensions that might interfere with authentication

Prevention: Keep your browser updated and avoid multiple Google accounts in the same browser session

Error: "Email already registered"

Symptoms: Registration fails for external users

Cause: Account already exists with the provided email

Solution:

  1. Try logging in instead of registering
  2. Use the "Forgot Password" option if available
  3. Check if you previously registered with Google OAuth

Session Management Errors

Error: "Session not found or expired"

Symptoms: Unable to proceed to calculated columns or dependency modeling

Cause: Session expired or browser cookies cleared

Solution:

  1. Return to Step 1 (Data File Checks) and re-upload your files
  2. Ensure cookies are enabled in your browser
  3. Avoid opening multiple tabs with AbhiStat
  4. Complete your analysis workflow within the session timeout period

Prevention: Work efficiently through the 4-step process without long breaks

Error: Session data inconsistency

Symptoms: Data appears different between steps or calculations fail unexpectedly

Cause: Browser storage conflicts or multiple session interference

Solution:

  1. Use the "Clear Data" button to reset your session
  2. Close all AbhiStat browser tabs and start fresh
  3. Clear browser cache and cookies for the AbhiStat domain
  4. Re-upload your files from the beginning

File Upload & Validation Errors

Error: "Invalid file type"

Symptoms: File upload rejected with format error

Cause: Unsupported file format uploaded

Solution:

  1. Convert your file to CSV, Excel (.xlsx/.xls), or Parquet format
  2. Ensure file extension matches the actual format
  3. For CSV files, use comma-separated values with proper encoding (UTF-8)

Supported Formats: .csv, .xlsx, .xls, .parquet

Error: "Unnamed columns detected"

Symptoms: File processing fails with unnamed column warning

Cause: Excel or CSV files contain columns without headers

Solution:

  1. Click "Remove Unnamed Columns" to automatically exclude them
  2. Or edit your source file to add proper column headers
  3. Ensure the first row contains column names, not data

Prevention: Always include descriptive headers in the first row of your data files

Error: "Mismatched columns detected"

Symptoms: Files have different column structures

Cause: The two uploaded files don't have identical column names

Solution:

  1. Click "Remove Mismatched Columns" to keep only common columns
  2. Or manually edit your files to ensure identical column names
  3. Check for spelling differences, extra spaces, or case sensitivity

Prevention: Use consistent column naming across both data files

Error: "Server error. Please try again."

Symptoms: File upload fails with generic server error

Cause: Network issues, server overload, or file corruption

Solution:

  1. Check your internet connection
  2. Try uploading smaller files (under 50MB recommended)
  3. Wait a few minutes and retry
  4. Try using a different browser or incognito mode
  5. Contact support if the issue persists

Formula Validation Errors

Error: "Column name can only contain letters, numbers and underscores"

Symptoms: Calculated column creation fails

Cause: Invalid characters in column name

Solution:

  1. Use only letters (a-z, A-Z), numbers (0-9), and underscores (_)
  2. Remove spaces, hyphens, or special characters
  3. Example: "New_Column_1" instead of "New Column-1"

Error: "Column name is too long (max 30 characters)"

Symptoms: Column name validation fails

Cause: Column name exceeds 30 character limit

Solution:

  1. Shorten your column name to 30 characters or less
  2. Use abbreviations or acronyms
  3. Example: "Avg_Monthly_Revenue" instead of "Average_Monthly_Revenue_Calculation"

Error: "Column name already exists in the original dataset"

Symptoms: Cannot create calculated column with existing name

Cause: Attempting to use a column name that already exists

Solution:

  1. Choose a different, unique column name
  2. Add a suffix like "_calc" or "_new"
  3. Example: "Revenue_Calc" instead of "Revenue"

Error: "Division by zero"

Symptoms: Formula execution fails during calculation

Cause: Formula attempts to divide by zero or empty values

Solution:

  1. Check your data for zero or null values in denominator columns
  2. Add conditional logic to handle zero values
  3. Clean your data to remove or replace zero values before upload

Prevention: Always validate your data for mathematical edge cases

Error: "Invalid expression" or "Mismatched parentheses"

Symptoms: Formula validation fails with syntax error

Cause: Incorrect formula syntax or unbalanced parentheses

Solution:

  1. Check that every opening parenthesis "(" has a matching closing ")"
  2. Ensure operators are placed between operands
  3. Verify column references are properly formatted
  4. Use the formula builder interface to avoid syntax errors

Example: "(Column_A + Column_B) * 2" instead of "Column_A + Column_B) * 2"

Error: "Invalid number: [value]"

Symptoms: Formula contains invalid numeric values

Cause: Non-numeric text entered where number expected

Solution:

  1. Use only valid numbers (integers or decimals)
  2. Use period (.) for decimal separator, not comma
  3. Example: "3.14" instead of "3,14" or "three"

Browser & Performance Issues

Issue: Slow loading or unresponsive interface

Symptoms: Pages load slowly or interface becomes unresponsive

Cause: Large datasets, browser limitations, or network issues

Solution:

  1. Close unnecessary browser tabs and applications
  2. Use a modern browser (Chrome, Firefox, Safari, Edge)
  3. Clear browser cache and cookies
  4. Reduce dataset size if possible (see performance optimization section)
  5. Ensure stable internet connection

Issue: Charts or visualizations not displaying

Symptoms: Blank areas where charts should appear

Cause: JavaScript disabled, browser compatibility, or data issues

Solution:

  1. Enable JavaScript in your browser settings
  2. Disable ad blockers or browser extensions temporarily
  3. Try refreshing the page
  4. Check if your data contains the required numeric columns
  5. Use a different browser to test

6.2 Performance Optimization & Best Practices

Data Preparation Guidelines

Optimal File Sizes

  • Recommended: Files under 50MB for best performance
  • Maximum: Files up to 200MB (may experience slower processing)
  • Rows: Optimal range is 1,000 to 100,000 rows
  • Columns: Keep under 50 columns when possible

File Format Recommendations

CSV Files
  • Best for: Simple data, fastest loading
  • Encoding: Use UTF-8 encoding
  • Separators: Use commas, avoid semicolons
  • Headers: Always include column headers in first row
Excel Files (.xlsx)
  • Best for: Formatted data with multiple sheets
  • Limitations: Slower processing than CSV
  • Tips: Remove formatting, formulas, and charts before upload
  • Sheets: Use simple sheet names without special characters
Parquet Files
  • Best for: Large datasets, fastest processing
  • Advantages: Compressed, optimized for analytics
  • Use case: When working with data science pipelines

Data Quality Optimization

  • Clean Data: Remove empty rows and columns before upload
  • Consistent Types: Ensure columns have consistent data types
  • Missing Values: Handle null/empty values appropriately
  • Numeric Data: Use proper numeric formats (no text in number columns)
  • Date Formats: Use consistent date formats (YYYY-MM-DD recommended)

Workflow Optimization Tips

Efficient Analysis Workflow

  1. Plan Ahead: Prepare your data files before starting the session
  2. Work Continuously: Complete the 4-step process without long breaks
  3. Save Progress: Use browser bookmarks to save your current step
  4. Document Formulas: Keep notes of calculated column formulas for future reference
  5. Export Results: Download charts and results immediately after generation

Browser Optimization

  • Recommended Browsers: Chrome, Firefox, Safari, Edge (latest versions)
  • Memory Management: Close unnecessary tabs and applications
  • Cache Management: Clear cache if experiencing issues
  • Extensions: Disable unnecessary browser extensions
  • JavaScript: Ensure JavaScript is enabled

Large Dataset Handling

For datasets over 100MB or 50,000 rows:
  • Sample Data: Consider using a representative sample for initial analysis
  • Filter Data: Remove unnecessary columns and rows before upload
  • Split Analysis: Break large datasets into smaller, manageable chunks
  • Patience: Allow extra time for processing and visualization
  • System Resources: Ensure adequate RAM (8GB+ recommended)

Session Management Best Practices

Maintaining Active Sessions

  • Single Tab: Use only one browser tab for AbhiStat
  • Cookies Enabled: Ensure browser cookies are enabled
  • Avoid Refreshing: Use navigation buttons instead of browser refresh
  • Complete Workflow: Finish the 4-step process in one session
  • Clear Data: Use "Clear Data" button when starting new analysis

Troubleshooting Session Issues

If you lose your session:
  1. Check if you can navigate back to previous steps
  2. Look for "Clear Data" button to reset cleanly
  3. Close all AbhiStat tabs and start fresh
  4. Clear browser cookies for the AbhiStat domain
  5. Re-upload your files from Step 1

Common Data Preparation Tasks

Cleaning Column Names

❌ Problematic Names:
  • "Revenue ($)" → Contains special characters
  • "Monthly Sales-2024" → Contains hyphens
  • "Customer Name" → Contains spaces
  • "%" → Too short and special character
✅ Corrected Names:
  • "Revenue_USD" → Uses underscore instead of special chars
  • "Monthly_Sales_2024" → Uses underscores instead of hyphens
  • "Customer_Name" → Uses underscore instead of space
  • "Percentage" → Descriptive name without special chars

Handling Missing Data

Recommended Approaches:
  • Numeric Data: Leave cells empty or use 0 if appropriate
  • Text Data: Leave cells empty or use "N/A" consistently
  • Dates: Leave cells empty rather than using placeholder dates
  • Avoid: Using "NULL", "n/a", "missing", or inconsistent placeholders

Numeric Data Formatting

Best Practices:
  • Decimal Separator: Use period (.) not comma (,)
  • Currency: Remove currency symbols ($, €, etc.)
  • Percentages: Use decimal format (0.15 instead of 15%)
  • Large Numbers: Remove thousand separators (1000000 not 1,000,000)
  • Scientific Notation: Convert to standard decimal format when possible

Excel-Specific Preparation

Before Uploading Excel Files:

  1. Remove Formatting: Clear cell colors, fonts, and borders
  2. Delete Formulas: Convert formulas to values (Paste Special → Values)
  3. Remove Charts: Delete any embedded charts or objects
  4. Clean Sheets: Remove empty sheets and rename remaining sheets simply
  5. Check Data Types: Ensure columns have consistent data types
  6. Remove Filters: Clear any active filters or sorting

Sheet Selection Tips:

  • Simple Names: Use sheet names like "Data1", "Data2" without spaces
  • Data Location: Ensure data starts from cell A1
  • Single Dataset: Each sheet should contain only one dataset
  • Consistent Structure: Both files should have data in the same sheet position

Validation Before Upload

Final Validation Checklist:

  1. Open Both Files: Verify they open correctly in Excel or a text editor
  2. Compare Structures: Ensure column names and order match exactly
  3. Check Sample Data: Verify first few rows contain expected data types
  4. Count Columns: Confirm both files have the same number of columns
  5. Test Upload: Try uploading a small sample first if working with large files
Pro Tip: Keep backup copies of your original data files before making any modifications for AbhiStat upload.
