Data Management

Learn how to upload, validate, and manage your data files in AbhiStat for statistical analysis.

Overview

Data management is the first and most crucial step in your AbhiStat workflow. This section covers everything you need to know about uploading files, understanding validation processes, and managing your data sessions effectively.

What You'll Learn

  • Supported file formats and requirements
  • Step-by-step file upload process
  • Data validation and preview features
  • Excel sheet selection capabilities
  • Session management and file lifecycle
  • Common troubleshooting scenarios

Supported File Formats

AbhiStat supports three main file formats for data upload:

CSV Files (.csv)

Best for: Simple tabular data with comma-separated values

  • Lightweight and fast to process
  • Universal compatibility
  • Ideal for large datasets
Example: data.csv, measurements.csv

Excel Files (.xlsx, .xls)

Best for: Structured data with multiple sheets or complex formatting

  • Support for multiple worksheets
  • Preserves data types and formatting
  • Interactive sheet selection
Example: analysis.xlsx, report.xls

Parquet Files (.parquet)

Best for: Large datasets requiring efficient storage and processing

  • Optimized for analytical workloads
  • Compressed file size
  • Fast read/write operations
Example: dataset.parquet, analytics.parquet

File Upload Process

AbhiStat requires two data files for comparative statistical analysis:

1

Navigate to Data Upload

From the AbhiStat dashboard, click on "Start Analysis" or navigate to the "Data File Checks" section. This is Step 1 of the four-step analysis workflow.

2

Upload "Without Product" File

Click on the left upload area labeled "Upload file for without product" or drag and drop your baseline/control data file.

  • This represents your baseline or control group data
  • Supported formats: CSV, Excel, or Parquet
  • File size limit: Reasonable for web processing
3

Upload "With Product" File

Click on the right upload area labeled "Upload file for with product" or drag and drop your treatment/experimental data file.

  • This represents your treatment or experimental group data
  • Must have compatible column structure with the first file
  • Same format requirements as the first file
4

File Processing and Validation

Once both files are uploaded, AbhiStat automatically processes and validates them:

  • Checks for compatible column structures
  • Validates data types and formats
  • Identifies and handles unnamed columns
  • Generates data preview for verification

Important Notes

  • Both files must be uploaded before processing begins
  • Files are processed simultaneously for column compatibility
  • Large files may take longer to process - please be patient
  • Ensure stable internet connection during upload

Excel Sheet Selection

When uploading Excel files with multiple worksheets, AbhiStat provides an interactive sheet selection interface:

How Sheet Selection Works

  1. Automatic Detection: AbhiStat automatically detects all available worksheets in your Excel file
  2. Sheet Selector: A dropdown menu appears showing all available sheets
  3. Default Selection: The first sheet is selected by default (marked as "default")
  4. Confirmation: Click "Confirm" to proceed with the selected sheet

Best Practices for Excel Files

  • Place your main data in the first worksheet for automatic selection
  • Use descriptive sheet names (e.g., "Raw Data", "Measurements")
  • Ensure data starts from the first row with column headers
  • Avoid merged cells or complex formatting in data areas

Data Validation and Preview

AbhiStat performs comprehensive validation to ensure your data is ready for analysis:

Validation Checks

Column Structure Compatibility

Ensures both files have matching column names and structures for comparative analysis.

Unnamed Column Detection

Identifies columns with names like "Unnamed: 0" that may cause processing issues.

Data Type Validation

Verifies that numeric columns contain valid numerical data for statistical calculations.

Missing Data Assessment

Identifies null values and provides guidance on handling missing data.

Data Preview Features

After successful validation, AbhiStat provides a comprehensive preview of your data:

  • File Information Summary: Shows file names, sizes, row counts, and column counts
  • Column List: Displays all column names from both files
  • Data Sample: Shows the first 10 rows of each dataset
  • Data Types: Indicates whether columns contain numeric or text data
  • Processing Status: Confirms successful validation and readiness for next steps

Common Validation Errors

Understanding and resolving common validation errors:

Unnamed Columns Error

Error Message: "Unnamed columns detected: File 'data.xlsx' has unnamed columns at indexes [0, 5]."

Cause: Excel files often have empty column headers that get named "Unnamed: 0", "Unnamed: 1", etc.

Solution: AbhiStat can automatically remove these columns, or you can fix them in your source file by adding proper column headers.

Mismatched Columns Error

Error Message: "Mismatched columns detected: Columns only in 'file1.csv': ['Temperature']. Columns only in 'file2.csv': ['Temp']."

Cause: The two files have different column names or structures.

Solution: Either standardize column names in your source files, or use AbhiStat's option to work with common columns only.

File Format Error

Error Message: "Invalid file type. Please upload a CSV, Excel, or Parquet file."

Cause: Uploaded file is not in a supported format.

Solution: Convert your file to CSV, Excel (.xlsx/.xls), or Parquet format before uploading.

File Reading Error

Error Message: "Error reading file: [specific error details]"

Cause: File may be corrupted, password-protected, or have formatting issues.

Solution: Verify file integrity, remove password protection, and ensure proper formatting.
